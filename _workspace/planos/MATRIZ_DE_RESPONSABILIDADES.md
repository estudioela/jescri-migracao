# MATRIZ DE RESPONSABILIDADES DOCUMENTAIS

> Status: Draft v1

Este documento define a responsabilidade oficial de cada documento canônico do Projeto Tear.

Cada documento deve responder exatamente uma pergunta.

Nenhum documento pode assumir responsabilidades pertencentes a outro.

---

# Princípios

- Um documento responde uma pergunta.
- Um conhecimento possui uma única fonte da verdade.
- Documentos referenciam outros documentos, nunca duplicam conteúdo.
- O Workspace nunca é documentação oficial.
- Histórico nunca descreve o estado atual do sistema.

---

# Responsabilidades Exclusivas

## README

Apresentação institucional do Projeto Tear.

---

## KNOWLEDGE_ARCHITECTURE

Define a arquitetura do conhecimento.

---

## PROJECT_PHILOSOPHY

Define os valores permanentes do projeto.

---

## ENGINEERING_PRINCIPLES

Define como decisões técnicas devem ser tomadas.

---

## SYSTEM_MAP

Representa a arquitetura vigente do sistema.

---

## KNOWN_DECISIONS

Consolida decisões permanentes aprovadas.

---

## AI_WORKFLOW

Define protocolos de trabalho para agentes de IA e colaboradores humanos.

---

## PROJECT_STATUS

Registra exclusivamente o estado atual do projeto.

---

## Regras de Validação

Uma alteração documental somente será considerada válida quando:

- não criar duplicação;
- respeitar esta matriz;
- preservar a fonte única da verdade;
- manter a responsabilidade exclusiva do documento.

---

# Critério de Conclusão

A Matriz de Responsabilidades será considerada estável quando todos os documentos oficiais possuírem uma responsabilidade exclusiva, claramente definida e sem sobreposição.
