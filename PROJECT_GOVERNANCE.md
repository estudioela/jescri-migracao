# PROJECT GOVERNANCE

## Estrutura oficial do projeto
- /mae → sistema principal (ERP)
- /sites → projetos auxiliares
- /docs → documentação
- /archive → histórico e backups
- /assets → recursos estáticos

## Regras obrigatórias
- Apenas 1 repositório Git (root)
- Proibido submodules
- Proibido .git internos
- Nada deve fugir dessa estrutura

## Regra de segurança
- Nunca apagar sem backup
- Mudanças estruturais devem ser justificadas
