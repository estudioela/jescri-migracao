- docs/SYSTEM_MAP.md
- docs/KNOWN_DECISIONS.md
- docs/CHANGELOG_DE_DESENVOLVIMENTO.md

Considere esses documentos como a fonte oficial do projeto.

---

# REGRAS

- Mantenha todos os testes verdes.
- Faça commits pequenos e atômicos.
- Reutilize a arquitetura existente.
- Não crie arquivos desnecessários.
- Não reabra decisões arquiteturais já aprovadas.
- Tome decisões técnicas locais sem solicitar confirmação quando não alterarem a arquitetura.
- Corrija apenas inconsistências que impeçam esta sprint.

---

# COMUNICAÇÃO

Durante a implementação:

Não apresente planos intermediários.

Não escreva documentação extensa.

Não faça relatórios.

Implemente.

Ao concluir cada bloco informe apenas:

- arquivos alterados;
- testes executados;
- resultado.

---

# CRITÉRIO DE CONCLUSÃO

A sprint termina quando:

- todos os objetivos forem concluídos;
- os testes permanecerem verdes;
- os commits forem realizados;
- não existirem alterações pendentes.

Após isso:

encerre a execução.

Não inicie melhorias não solicitadas.
